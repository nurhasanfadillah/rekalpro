# ReKal Mobile & PWA Implementation Guide

## 🎯 Features Implemented

### 1. Progressive Web App (PWA)
- **Web App Manifest**: Configured with app name, icons, theme colors
- **Service Worker**: Auto-generated by `vite-plugin-pwa` for offline caching
- **Installable**: Can be added to home screen on iOS/Android
- **Standalone Mode**: Runs like a native app without browser chrome

### 2. Mobile Native UI/UX
- **Bottom Navigation**: Native-style bottom nav bar on mobile devices
- **Touch Feedback**: Active states, scale animations on buttons
- **Safe Area Support**: Handles notched phones (iPhone X+, Android)
- **Optimized Touch Targets**: Minimum 44px touch targets
- **Mobile-First Cards**: Responsive card layouts for mobile lists
- **Offline Indicator**: Visual indicator when app is offline

### 3. LocalStorage & Offline Support
- **Data Caching**: Products, Materials, Categories cached locally
- **Offline Mode**: App works without internet using cached data
- **Auto-Refresh**: Data refreshes automatically when coming back online
- **Cache Duration**: 5-minute cache with automatic invalidation
- **Pending Changes**: Queue system for offline edits (future enhancement)

## 📱 Mobile Experience

### Navigation
- **Desktop**: Sidebar navigation with collapse option
- **Mobile**: Bottom tab bar with 5 quick actions:
  - Beranda (Dashboard)
  - Produk (Products)
  - Material (Materials)
  - Kategori (Categories)
  - Tambah (Quick Add)

### Visual Indicators
- **Offline Banner**: Amber banner at top when offline
- **Cached Data Badge**: Blue info box when showing cached data
- **Refresh Button**: Mobile-optimized refresh in header

### Touch Optimizations
- Button press animations (`active:scale-95`)
- Touch highlight removal (`-webkit-tap-highlight-color: transparent`)
- Prevent zoom on input focus (`text-base` on mobile inputs)
- Smooth scrolling enabled

## 🚀 How to Use

### Installing the PWA

#### iOS (Safari):
1. Open the app in Safari
2. Tap the Share button (square with arrow)
3. Scroll down and tap "Add to Home Screen"
4. Tap "Add"

#### Android (Chrome):
1. Open the app in Chrome
2. Tap the menu (3 dots)
3. Tap "Add to Home screen" or "Install app"
4. Tap "Install"

#### Desktop (Chrome/Edge):
1. Look for install icon in address bar
2. Click "Install ReKal"

### Offline Usage
1. **First Visit**: App loads data from server and caches it
2. **Subsequent Visits**: 
   - Online: Shows cached data immediately, refreshes in background
   - Offline: Shows cached data with offline indicator
3. **Going Offline**: App automatically detects and shows offline banner
4. **Coming Back Online**: App automatically refreshes data

### Refreshing Data
- **Desktop**: Data refreshes automatically on page load
- **Mobile**: Tap the refresh button in the header or pull-to-refresh (if implemented)

## 🎨 Customization

### Changing Theme Colors
Edit `vite.config.js`:
```javascript
manifest: {
  theme_color: '#your-color',      // Status bar color
  background_color: '#your-color',  // Splash screen background
}
```

### Adding More Icons
1. Generate icons at: https://www.pwabuilder.com/imageGenerator
2. Place in `client/public/icons/`
3. Update `manifest.json` with new icon paths

### Adjusting Cache Duration
Edit `client/src/utils/storage.js`:
```javascript
const CACHE_DURATION = 10 * 60 * 1000; // 10 minutes
```

## 🔍 Testing

### PWA Validation
1. Open Chrome DevTools → Lighthouse
2. Run "PWA" audit
3. Check for 100% score

### Offline Testing
1. Open DevTools → Network tab
2. Set "Throttling" to "Offline"
3. Refresh page - should show cached data
4. Check for offline indicator banner

### Mobile Testing
1. Open DevTools → Toggle Device Toolbar
2. Select iPhone/Android preset
3. Test:
   - Bottom navigation visibility
   - Touch feedback on buttons
   - Card layouts
   - Safe area padding (notch simulation)

## 🐛 Troubleshooting

### Icons Not Showing
- Ensure PNG files exist in `public/icons/`
- Check file naming matches manifest
- Clear browser cache and reload

### Offline Not Working
- Check Service Worker registered in DevTools → Application → Service Workers
- Verify `sw.js` exists in `dist/` folder after build
- Check console for SW errors

### Mobile Layout Issues
- Verify `viewport-fit=cover` in index.html
- Check `safe-top` and `safe-bottom` classes applied
- Test on actual device (simulators may not show notches)
